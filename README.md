# Cisco Packet Tracer - Basic Network Security Lab

## 📌 Project Overview

This project is a basic networking lab created using **Cisco Packet Tracer** as part of my CCNA learning journey.

The lab demonstrates a simple enterprise-style network topology with two branches, routers, switches, firewalls, PCs, servers, and an attacker laptop for basic security testing concepts.

The main purpose of this project is to understand how different network devices are connected and how traffic flows between branch networks through routers, switches, and firewalls.

---

## 🖥️ Network Topology

The topology includes:

- New York Branch
- Tokyo Branch
- Internet Router
- Cisco Routers
- Cisco Switches
- Cisco ASA Firewalls
- End Devices
- Servers
- Attacker Laptop for testing/security scenario

## 🎯 Lab Objectives

The main objectives of this lab are:

- Create a basic network topology in Cisco Packet Tracer
- Connect routers, switches, firewalls, PCs, and servers
- Understand branch-to-branch network connectivity
- Practice Cisco device configuration
- Learn basic network security design
- Use an attacker laptop to understand security testing concepts
- Build practical networking and CCNA skills

---

## 🌐 Topology Description

The lab contains two main branches:

### New York Branch

The New York branch contains:

- 2 PCs
- 1 Cisco 2960 switch
- 1 Cisco 2911 router
- 1 Cisco ASA firewall

### Tokyo Branch

The Tokyo branch contains:

- 2 servers
- 1 Cisco 2960 switch
- 1 Cisco 2911 router
- 1 Cisco ASA firewall

### Internet Section

The middle router represents the internet connection between the two branches.

An attacker laptop is also connected to the internet router to represent a basic security testing scenario.

---

## 🔐 Security Concepts Practiced

This lab can be used to practice:

- Router password configuration
- Console password configuration
- Enable secret password
- Basic firewall placement
- Network segmentation
- Secure traffic flow between branches
- Basic attacker simulation concept
